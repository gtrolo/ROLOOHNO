## AI Backend — Uncensored Content Generatie

Mainstream AI-providers (OpenAI, Google, Anthropic) hebben expliciete content op API- en consumentenniveau volledig geblokkeerd. Voor een app die live expliciete opdrachten genereert zijn de volgende alternatieven beschikbaar.

### 1. Kant-en-klare Web/Apps

| Platform | Omschrijving |
|---|---|
| **NovelAI** | Gebouwd voor ongecensureerde creatieve teksten. Eigen getrainde modellen zonder filters. |
| **Backyard AI** (voorheen Faraday.dev) | Interface voor ongecensureerde modellen, lokaal of via cloud. Gebouwd voor expliciete roleplay. |
| **Chub.ai** | Volledig gericht op NSFW-interacties, vaak in combinatie met uncensored API's. |

### 2. API-route voor App Backend

| Optie | Omschrijving |

| **OpenRouter** | API-aggregator met honderden open-source modellen. Veiligheidsfilters uitzetten via accountinstellingen. Modellen zoals *MythoMax* of *Nous Hermes* beschikbaar. |
| **RunPod / vast.ai** | Huur een cloud-GPU per uur en host een uncensored LLM zelf. Volledig eigen API-endpoint, geen externe filtering. |

Download **LM Studio** of **Ollama** en haal via Hugging Face een uncensored model op:

## Consent-Gated Hardcore Engine — Architectuur Blauwdruk

Een gefuseerde aanpak: robuuste consent- en datastructuur gecombineerd met een brute, ongecensureerde actie-gedrevenheid. Geen saaie keuze-menu's tijdens de actie — categorieën worden uitsluitend aan de voorkant gebruikt als filter. Zodra de deur open is, neemt de ongecensureerde engine het over met expliciete directieven.

### 1. Architectuur: Categorieën als Brandstof, Niet als Rem

Abstracte categorieën worden uitsluitend gebruikt in de set-up fase en als waarschuwingssysteem. Dit bouwt psychologische spanning op en garandeert 100% consent, zonder dat de uiteindelijke actie verwatert.

- **Profiel Set-up:** Bij het inloggen vinken spelers hun "Zones" aan (bijv. Oraal, Bondage, Penetratie, Voyeurisme). Dit is de datalaag waar de app wiskundig mee rekent.
- **De Consent Gate:** Voordat een zware opdracht wordt getoond, toont de app alleen de categorie en de betrokken spelers op het hoofdscherm.
- **De Blind Swipe:** De betrokken spelers swipen op hun eigen telefoon akkoord met de categorie, zonder te weten wat de exacte opdracht is. Dit creëert maximale anticipatie.

### 2. Generatie Engine: Directieven, Geen Keuzemenu's

Zodra de Consent Gate is gepasseerd, genereert de ongecensureerde AI (of de hardcoded expliciete database) een genadeloos directief.

| Fase | Brave Methode | Gefuseerde Methode |
|---|---|---|
| **Aankondiging** | "A private oral-category card is available." | Hoofdscherm flitst rood: "Speler A & B. Categorie: Oraal. 60 Seconden." |
| **Actie op Telefoon** | "Choose: receive / give / discuss / skip." | Speler A & B swipen 'Accept'. Spanning bouwt op. |
| **De Executie** | Spelers overleggen samen wat ze gaan doen. | Scherm toont direct een ongecensureerde, expliciete opdracht via Uncensored API of hardcoded database. |

### 3. Systeem & Tech Integratie

| Laag | Technologie | Rol |
|---|---|---|
| **Frontend** | Next.js & Framer Motion | Beheert de visuele Consent Gates, toont abstracte iconen/categorieën, haptische feedback bij grensacceptatie |
| **Backend Database** | Supabase | Per speler een JSON-object met geaccepteerde tags: `["kissing", "spanking", "oral", "public"]` |
| **Matchmaker** | Backend logica | Zoekt constant naar overlappende tags tussen willekeurige spelers in de kamer |
| **Uncensored Output** | OpenRouter / Eigen Database | Ontvangt ping met spelers + tag + intensiteitsniveau, stuurt rauwe tekst terug na Consent Swipe |

**Voorbeeld Matchmaker ping:**
```
Maak een opdracht voor Speler A en B binnen de tag "oral", intensiteit niveau 8.
```

### 4. Veto & Safe-Word Systeem

Vermomd als game-element, maar keihard veiligheidsmechanisme.

- **De Veto Token:** Elke speler krijgt 2 Veto-tokens zichtbaar op het scherm. Inzetten skipt de opdracht direct, maar kost punten of resulteert in een lichtere straf.
- **Panic Button:** Altijd zichtbaar, puur zwart op donkere achtergrond. Één druk pauzeert het hele spel op alle verbonden schermen direct, zonder vragen.

### 5. Asynchrone Escalatie (De Achtergrond Game)

Naast de hoofdflow draait de categorie-logica door op de achtergrond, volledig onafhankelijk van het hoofdscherm.

**Voorbeeld:**
- De app detecteert dat Speler C de categorie "Voyeurisme" aan heeft staan.
- Speler C krijgt een privébericht: *"Ga in de hoek van de kamer staan. Kijk 5 minuten lang naar de persoon in de rode stoel. Raak jezelf aan."*
- Dit gebeurt volledig parallel aan wat er op het hoofdscherm speelt.

> Het systeem regelt de grenzen onzichtbaar wiskundig op de achtergrond, zodat gebruikers aan de voorkant pure, ongefilterde chaos en escalatie ervaren.

> **Role:** Elite Next.js, Supabase, and UI/UX developer building a high-arousal, real-time multiplayer PWA for local group dynamics. Core mechanic: **Consent-Gated Action Engine**.

### Core Logic & Flows (Consent-Gated Engine)

1. **Lobby & Sync** — Host creates a room. Clients join via 4-digit `room_code`. All screens sync instantly via Supabase Realtime.
2. **Profile Setup (Silent)** — Players select `consented_tags` on join. Data saved to Supabase, hidden from other players.
3. **The Matchmaker (Backend/Edge)** — Utility reads all players in the room, finds overlapping `consented_tags` between 2+ players, selects a target category.
4. **The Consent Gate (UI)**
   - Host screen flashes red.
   - Displays: *"Players A & B. Category: [Matched Tag]."*
   - Players A & B receive a swipe prompt: Right = Accept, Left = Veto.
5. **The Uncensored Execution**
   - If both accept → frontend calls `/api/generate-action`.
   - Endpoint connects to Uncensored LLM via OpenRouter (or fetches from local JSON database).
   - Returned action displayed in raw text on main screen.

### UI/UX Critical Requirements

- **Web Haptics API** — `navigator.vibrate` fires rhythmically during Consent Gate waiting period.
- **Swipe Physics** — Framer Motion `useAnimation` + `PanInfo` for fluid physical swipe cards.
- **Panic Button** — Persistent minimal black button on every screen. One press broadcasts `PAUSE` via Supabase, overlaying black screen with `"SCENE PAUSED"` on all connected devices.
